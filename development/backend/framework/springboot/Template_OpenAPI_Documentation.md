# OpenAPI 3.0 Specification Template & Documentation Standards

---

## 1. OpenAPI 3.0 Specification Template

```yaml
openapi: 3.0.3
info:
  title: Transfer Service API
  description: |
    Domestic fund transfer service for retail and premium banking customers.
    Supports transfer initiation, status tracking, and cancellation.
  version: 1.0.0
  contact:
    name: Payment Team
    email: payment-team@bank.com
  license:
    name: Internal Use Only

servers:
  - url: https://api.bank.com/v1
    description: Production
  - url: https://api-staging.bank.com/v1
    description: Staging
  - url: http://localhost:8080/v1
    description: Local development

tags:
  - name: Transfers
    description: Fund transfer operations

security:
  - bearerAuth: []

paths:
  /transfers:
    post:
      tags: [Transfers]
      operationId: initiateTransfer
      summary: Initiate domestic fund transfer
      description: |
        Initiates a domestic fund transfer from the authenticated customer's account
        to a registered beneficiary. Requires step-up MFA. Subject to daily transfer limits.
        
        **Authentication:** Bearer JWT + MFA token
        **Authorization:** Scope `transfers:write`; source account must belong to authenticated customer
        **Idempotency:** Idempotency-Key header required
        **Rate Limit:** 10 requests/minute per user
      parameters:
        - $ref: '#/components/parameters/CorrelationId'
        - $ref: '#/components/parameters/IdempotencyKey'
        - $ref: '#/components/parameters/MfaToken'
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/TransferRequest'
            examples:
              domestic:
                summary: Standard domestic transfer
                value:
                  sourceAccountId: "ACC-001234"
                  beneficiaryId: "BEN-005678"
                  amount: 5000.00
                  currency: "USD"
                  reference: "Invoice 2026-001"
      responses:
        '201':
          description: Transfer initiated successfully
          headers:
            X-Correlation-ID:
              $ref: '#/components/headers/CorrelationId'
            X-Request-ID:
              $ref: '#/components/headers/RequestId'
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/TransferSuccessResponse'
        '400':
          $ref: '#/components/responses/ValidationError'
        '401':
          $ref: '#/components/responses/Unauthorized'
        '403':
          $ref: '#/components/responses/Forbidden'
        '409':
          $ref: '#/components/responses/DuplicateIdempotencyKey'
        '422':
          $ref: '#/components/responses/BusinessError'
        '429':
          $ref: '#/components/responses/RateLimitExceeded'
        '500':
          $ref: '#/components/responses/InternalError'

  /transfers/{transferId}:
    get:
      tags: [Transfers]
      operationId: getTransferStatus
      summary: Get transfer status
      description: |
        Retrieves the current status of a transfer. Only accessible by the transfer initiator.
        
        **Authorization:** Scope `transfers:read`; transfer must belong to authenticated customer
      parameters:
        - $ref: '#/components/parameters/TransferId'
        - $ref: '#/components/parameters/CorrelationId'
      responses:
        '200':
          description: Transfer details
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/TransferSuccessResponse'
        '401':
          $ref: '#/components/responses/Unauthorized'
        '403':
          $ref: '#/components/responses/Forbidden'
        '404':
          $ref: '#/components/responses/NotFound'

components:
  securitySchemes:
    bearerAuth:
      type: http
      scheme: bearer
      bearerFormat: JWT
      description: OAuth 2.0 JWT access token

  parameters:
    CorrelationId:
      name: X-Correlation-ID
      in: header
      required: false
      description: Correlation ID for distributed tracing. Generated by gateway if not provided.
      schema:
        type: string
        format: uuid
    IdempotencyKey:
      name: Idempotency-Key
      in: header
      required: true
      description: Client-generated UUID for duplicate prevention. Valid for 24 hours.
      schema:
        type: string
        format: uuid
    MfaToken:
      name: X-MFA-Token
      in: header
      required: true
      description: Step-up MFA verification token.
      schema:
        type: string
    TransferId:
      name: transferId
      in: path
      required: true
      description: Transfer identifier
      schema:
        type: string
        format: uuid

  headers:
    CorrelationId:
      description: Correlation ID for tracing
      schema:
        type: string
        format: uuid
    RequestId:
      description: Unique request identifier
      schema:
        type: string
        format: uuid

  schemas:
    TransferRequest:
      type: object
      required:
        - sourceAccountId
        - beneficiaryId
        - amount
        - currency
      properties:
        sourceAccountId:
          type: string
          description: Source account ID. Must belong to authenticated customer.
          example: "ACC-001234"
          pattern: "^ACC-[0-9]{6}$"
        beneficiaryId:
          type: string
          description: Registered beneficiary ID. Must be active.
          example: "BEN-005678"
          pattern: "^BEN-[0-9]{6}$"
        amount:
          type: number
          format: double
          description: Transfer amount. Positive, within daily limit.
          minimum: 0.01
          maximum: 999999999.99
          example: 5000.00
        currency:
          type: string
          description: ISO 4217 currency code
          enum: [USD, EUR, GBP, CHF, JPY]
          example: "USD"
        reference:
          type: string
          description: Optional payment reference
          maxLength: 140
          pattern: "^[a-zA-Z0-9 \\-/.]*$"
          example: "Invoice 2026-001"

    TransferSuccessResponse:
      type: object
      properties:
        data:
          $ref: '#/components/schemas/TransferData'
        meta:
          $ref: '#/components/schemas/Meta'

    TransferData:
      type: object
      properties:
        transferId:
          type: string
          format: uuid
          example: "TRF-2026-0315-001"
        status:
          type: string
          enum: [INITIATED, PROCESSING, COMPLETED, FAILED, CANCELLED]
          example: "INITIATED"
        amount:
          type: number
          example: 5000.00
        currency:
          type: string
          example: "USD"
        sourceAccount:
          type: string
          description: Masked account number (last 4 digits)
          example: "****1234"
        beneficiary:
          type: string
          description: Masked beneficiary reference
          example: "****5678"
        initiatedAt:
          type: string
          format: date-time
          example: "2026-03-15T15:30:00Z"
        completedAt:
          type: string
          format: date-time
          nullable: true

    ErrorResponse:
      type: object
      properties:
        error:
          type: object
          properties:
            code:
              type: string
              description: Machine-readable error code
              example: "VALIDATION_ERROR"
            message:
              type: string
              description: Human-readable error message
              example: "Request validation failed"
            details:
              type: array
              items:
                $ref: '#/components/schemas/FieldError'
        meta:
          $ref: '#/components/schemas/Meta'

    FieldError:
      type: object
      properties:
        field:
          type: string
          example: "amount"
        message:
          type: string
          example: "Must be greater than 0"
        rejectedValue:
          example: -100

    Meta:
      type: object
      properties:
        requestId:
          type: string
          format: uuid
        timestamp:
          type: string
          format: date-time

  responses:
    ValidationError:
      description: Request validation failed
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/ErrorResponse'
          example:
            error:
              code: "VALIDATION_ERROR"
              message: "Request validation failed"
              details:
                - field: "amount"
                  message: "Must be greater than 0"
                  rejectedValue: -100
            meta:
              requestId: "req-abc-123"
              timestamp: "2026-03-15T15:30:00Z"
    Unauthorized:
      description: Authentication required
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/ErrorResponse'
          example:
            error:
              code: "AUTH_REQUIRED"
              message: "Authentication required"
    Forbidden:
      description: Access denied
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/ErrorResponse'
          example:
            error:
              code: "ACCESS_DENIED"
              message: "Access denied"
    NotFound:
      description: Resource not found
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/ErrorResponse'
    DuplicateIdempotencyKey:
      description: Duplicate idempotency key with different request body
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/ErrorResponse'
    BusinessError:
      description: Business rule violation
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/ErrorResponse'
          examples:
            insufficientFunds:
              value:
                error:
                  code: "INSUFFICIENT_FUNDS"
                  message: "Insufficient funds for this transfer"
            limitExceeded:
              value:
                error:
                  code: "LIMIT_EXCEEDED"
                  message: "Daily transfer limit exceeded. Remaining: $5,000.00"
    RateLimitExceeded:
      description: Rate limit exceeded
      headers:
        Retry-After:
          schema:
            type: integer
          description: Seconds to wait before retrying
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/ErrorResponse'
    InternalError:
      description: Internal server error
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/ErrorResponse'
          example:
            error:
              code: "INTERNAL_ERROR"
              message: "Something went wrong. Please try again."
```

## 2. OpenAPI Documentation Standards

| Standard | Requirement |
|---|---|
| Spec location | src/main/resources/openapi/{service}-api.yaml |
| Version control | Spec committed alongside code; changes in same PR |
| Validation | Spectral lint on every build; invalid spec blocks merge |
| Breaking change detection | oasdiff in CI; alert on breaking changes |
| Code generation | SpringDoc generates spec from annotations OR spec-first with openapi-generator |
| Interactive docs | Swagger UI at /swagger-ui.html (staging only; disabled in production) |
| Examples | Every schema must have realistic examples with masked banking data |
| Error responses | Every endpoint documents all possible error codes with examples |
| Security | Security scheme documented per endpoint; scopes specified |
| Deprecation | Deprecated endpoints marked with deprecated: true + description of replacement |

### SpringDoc Configuration

```java
@Configuration
public class OpenApiConfig {
    @Bean
    public OpenAPI bankingOpenAPI() {
        return new OpenAPI()
            .info(new Info()
                .title("Transfer Service API")
                .version("1.0.0")
                .contact(new Contact().name("Payment Team").email("payment-team@bank.com")))
            .addSecurityItem(new SecurityRequirement().addList("bearerAuth"))
            .components(new Components()
                .addSecuritySchemes("bearerAuth",
                    new SecurityScheme()
                        .type(SecurityScheme.Type.HTTP)
                        .scheme("bearer")
                        .bearerFormat("JWT")));
    }
}
```

```yaml
# application.yml
springdoc:
  api-docs:
    path: /v3/api-docs

    path: /swagger-ui.html
    enabled: ${SWAGGER_UI_ENABLED:false}  # Disabled in production
  show-actuator: false
  default-produces-media-type: application/json
```

### Error Code Reference Template

```markdown
# Transfer Service — Error Codes

| Code | HTTP Status | Description | Resolution |
|---|---|---|---|
| VALIDATION_ERROR | 400 | Request validation failed | Check field-level details in response |
| AUTH_REQUIRED | 401 | Missing or invalid authentication | Provide valid Bearer JWT token |
| AUTH_TOKEN_EXPIRED | 401 | Token has expired | Refresh token and retry |
| ACCESS_DENIED | 403 | Not authorized for this resource | Verify account ownership and scopes |
| NOT_FOUND | 404 | Transfer not found | Verify transfer ID |
| DUPLICATE_REQUEST | 409 | Idempotency key already used | Use original response or new key |
| INSUFFICIENT_FUNDS | 422 | Account has insufficient balance | Check balance before transfer |
| LIMIT_EXCEEDED | 422 | Daily transfer limit exceeded | Check remaining limit |
| ACCOUNT_FROZEN | 422 | Source account is frozen | Contact support |
| FRAUD_HOLD | 422 | Transfer held for fraud review | Wait for review notification |
| RATE_LIMIT_EXCEEDED | 429 | Too many requests | Wait per Retry-After header |
| INTERNAL_ERROR | 500 | Unexpected server error | Retry; contact support if persistent |
| SERVICE_UNAVAILABLE | 503 | Dependency unavailable | Retry after delay |
```
