# PSR — Payment Systems Regulator

## Overview
| Field | Value |
|---|---|
| Full Name | Payment Systems Regulator |
| Jurisdiction | United Kingdom |
| Role | Economic regulator of payment systems in the UK |
| Established | 1 April 2015 under Financial Services (Banking Reform) Act 2013 |
| Penalties | Unlimited fines; directions; compliance orders |
| Scope | Operators, payment service providers, and infrastructure providers of designated payment systems |

## Designated Payment Systems
| System | Type | Operator |
|---|---|---|
| Faster Payments | Real-time payments | Pay.UK |
| Bacs | Direct debits, direct credits | Pay.UK |
| CHAPS | High-value, same-day | Bank of England |
| LINK | ATM network | LINK Scheme |
| Visa | Card payments | Visa Europe |
| Mastercard | Card payments | Mastercard |

## Key Regulatory Areas for Banking IT
| Area | Description | IT Implication |
|---|---|---|
| APP Fraud Reimbursement | Mandatory reimbursement for APP fraud victims (from Oct 2024) | Fraud detection systems; reimbursement workflows; Confirmation of Payee |
| Confirmation of Payee (CoP) | Name-checking service for payments | CoP API integration; match/no-match/partial-match handling |
| Access to payment systems | Fair and open access | API standards; interoperability |
| Interchange fee regulation | Cap on card interchange fees | Fee calculation systems |
